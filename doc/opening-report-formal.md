# 课题开题：面向离线场景的轻量级多模态提示注入检测算法研究
**课题类型**：实验性 / 算法研究  
**文档版本**：v1.2  

---

## 摘要

随着大语言模型（LLM）与多模态大语言模型（MLLM）的产业化部署，"提示注入"（Prompt Injection）已成为首要安全威胁。**业界主流防御方案**采用"在线 + 大模型"形态（云端 API、专用防御大模型、对齐训练），在**离线边缘场景**（隐私敏感行业、嵌入式设备、低成本合规审计）下面临网络依赖、隐私泄露、算力受限等问题。

本课题研究**面向离线场景的轻量级多模态提示注入检测算法**。其核心定位是：在 LLM 入口处构建一个**轻量、可打包、跨平台、无网络依赖**的前置过滤器。技术上采用**级联式方案**——先用 OCR 抽取图像中的文字，再与用户 prompt 拼接后送入多语种文本分类器（XLM-RoBERTa + LoRA）做 3 分类（clean / direct / indirect）。研究目标设备为**普通 PC 与主流边缘设备**，明确不覆盖 MCU 级极端嵌入式。

研究内容覆盖威胁模型、级联式检测算法、多语种数据构造、跨平台训练验证、离线部署特性测试。预期产出研究报告、开源训练代码、LoRA 权重、自构造数据集与离线打包示例。

---

## 一、研究背景与现状

### 1.1 攻击形态

**提示注入**指攻击者通过精心构造的输入，使 LLM "忽略"原始系统指令，转而执行攻击者意图。OWASP 基金会在 2023、2024 连续两年的 *Top 10 for LLM Applications* 中，将其列为**第一大风险**[1]。

**攻击形态已演进为三个分支**：
- **直接注入**：用户 prompt 中显式包含劫持指令，Perez & Ribeiro[2] 首次系统提出 *"Ignore previous instructions..."* 范式；
- **间接注入**：通过 LLM 应用检索的外部内容（网页、文档、邮件、图像）注入指令，由 Greshake 等[3] 首次系统命名；
- **多模态注入**：图像中嵌入 OCR 可读的攻击文本或对抗扰动。Qi 等[4] 证明视觉对抗可绕过对齐；Luo 等[5] 构建的 JailbreakV-28K 数据集首次大规模标注了多模态越狱样本。

### 1.2 防御路线与研究空白

**防御研究主要沿三条路线**展开：

| 路线 | 代表 | 形态特征 | 局限 |
|---|---|---|---|
| 训练时对齐 | StruQ[6]、SecAlign | 模型层加固 | 间接注入鲁棒性弱；需重训 |
| 大模型检测 | PromptGuard[7]、Lakera Guard[8] | **在线 API**，参数量大 | 算力大、网络依赖、隐私上传 |
| 轻量级检测 | 关键词规则、小型文本分类器 | 本地 | 多模态缺位、绕过高发 |

**离线边缘场景**（医疗/法律/政府的"数据不出端"、移动与 IoT 设备、中小企业低成本合规审计、金融与工控的高敏感流程节点）对防御方案的核心约束是**无网络依赖、小模型/轻算力、可打包分发、可解释**。现有研究在该象限存在以下不足：
1. 主流方案默认部署在云端，难以迁移到边缘；
2. 轻量检测器多仅处理文本，对图像内嵌指令覆盖不足；
3. 多语种（特别是中英混合）覆盖有限；
4. 模型大小、内存峰值、冷启动延迟等离线关键指标在论文中鲜有报告。

### 1.3 课题定位

综合上述，本课题拟研究的核心问题是：
> 在离线边缘部署的算力、内存、网络约束下，**如何构建一个轻量、可打包、跨平台的多模态提示注入检测算法**？

定位上聚焦三个差异化：
- 与业界"在线大模型"防御路线**互补而非竞争**；
- 在"前置过滤"这一**独立范式**上做算法级深耕；
- 在 XLM-RoBERTa + OCR 的级联方案之上，叠加**早退机制、零宽字符检测、跨模态矛盾检测**三项算法优化（详见 §3.6）。

## 二、研究目标与内容

### 2.1 总目标

研究一种**面向离线场景的轻量级多模态提示注入检测算法**。

**关键属性**：
1. **离线可打包**：模型权重 + 依赖 + 推理脚本可单一目录分发，**零网络依赖**；
2. **轻量**：默认骨干 ≤ 500M 参数，可在普通 PC CPU 上完成训练与推理；
3. **多模态**：同时处理图像与文本输入，覆盖直接 / 间接注入两类威胁；
4. **多语种**：以中、英为主，基于 XLM-RoBERTa[9] 支持 100+ 语种；
5. **跨平台**：macOS（Apple Silicon, MPS）与 x86 PC（CPU）上均可训练与推理。

### 2.2 具体研究内容

| 编号 | 研究内容 | 预期产出 |
|---|---|---|
| C1 | **多模态注入威胁模型构建** | 攻击分类、形式化定义、典型样例 |

| **C7** | **算法优化一：早退机制 (Early Exit)** | XLM-RoBERTa 中间层分类头 + 联合训练 + 自适应推理路径 |
| **C8** | **算法优化二：注入威胁类型扩展** | 零宽字符 / Unicode 混淆检测 + 跨模态矛盾检测（CLIP 相似度特征） |

为避免研究范围无限扩大，**显式声明以下边界**：

| 在范围内 | 不在范围内 |
|---|---|
| 离线场景、轻量模型、多模态、跨平台 | 在线服务、API 化部署 |
| 普通 PC、主流边缘设备 | MCU 级（< 1MB 内存）极端嵌入式 |

| 中、英及多语种覆盖 | 极致多语言对齐（只测覆盖度） |

## 三、研究方法与技术路线

### 3.1 研究方法

- **文献调研法**：系统梳理 2022 年以来提示注入攻防代表性工作；
- **数据驱动实验法**：基于自构造 + 公开数据集进行训练与消融；
- **对比实验法**：与 PromptGuard-86M、关键词基线做定量对比；
- **跨平台验证法**：在 macOS 与 x86 PC CPU 上分别训练并对比结果一致性；
- **离线部署特性测试法**：量化包大小、冷启动、内存峰值、网络流量等离线关键指标。

### 3.2 总体技术路线

```
                用户输入（图片 + 文本 prompt）
                          │
                          ▼
              ┌─────────────────────┐
              │  OCR 引擎（离线）    │  抽出图片内文字
              └──────────┬──────────┘
                          │
                          ▼
              [OCR 文本] ⊕ [用户 prompt]
                          │
                          ▼
              ┌─────────────────────┐
              │ XLM-RoBERTa + LoRA  │  3 分类
              │   (多语种分类器)     │  clean / direct / indirect
              └──────────┬──────────┘
                          │
                          ▼
                  [风险分 + 标签 + 可疑片段]
                          │
                          ▼
              通过 → 喂给真正的大模型
              拦截 → "⚠️ 检测到可疑输入"
```

### 3.3 关键技术

#### 3.3.1 OCR 引擎
- **主选**：`paddleocr`（中文识别效果好，模型可本地化）；
- **备选**：`pytesseract`（依赖更轻，跨平台更稳）；
- **离线打包**：OCR 模型与语言包必须包含在分发目录中。

#### 3.3.2 文本分类器
- **骨干**：`xlm-roberta-base`（约 2.7 亿参数，支持 100+ 语种）[9]；
- **微调方式**：LoRA（Low-Rank Adaptation）[10]，只调整约 1% 的参数，节省算力；
- **输出**：3 分类头（clean / direct / indirect）+ 风险分（0~1）。

> **术语简释**
> - **LoRA**：一种"小改动"训练技术——冻结原始模型参数，只训练两个小矩阵来近似权重变化，使显存和算力需求大幅下降。详见 [10]。
> - **OCR**：Optical Character Recognition，光学字符识别，即把图片中的文字识别为机器可读文本。

#### 3.3.3 数据流
- 输入：(图像, 用户 prompt) 对；
- 步骤 1：OCR 抽出图像内文字；
- 步骤 2：`[OCR 文本] + [SEP] + [用户 prompt]` 拼接为单序列；
- 步骤 3：送入 XLM-RoBERTa，取 `[CLS]` 隐藏态 → 分类头 → 3 分类 logits；
- 步骤 4：softmax 得到概率分布，取 argmax + 风险分。

### 3.4 离线部署设计原则

针对离线边缘场景的工程约束，框架设计遵循以下原则：

1. **零网络依赖**：运行时禁止任何 HTTP/外部 IO；OCR 引擎与 backbone 全部本地权重；
2. **可打包分发**：训练完成后产出单一 `runs/<run_id>/artifacts/package/mpid_offline/` 目录；
3. **推理引擎可替换**：默认 PyTorch，可选 ONNX Runtime（CPU 推理加速）；
4. **跨平台**：macOS Apple Silicon 与 x86 PC CPU 均验证。

### 3.5 跨平台策略

- 所有张量操作禁止 hard-code `.cuda()`；
- 统一使用 `torch.device("mps" if torch.backends.mps.is_available() else "cpu")`；
- OCR 引擎在 mac 与 x86 上均提供预编译 wheel；
- 训练脚本通过 `--device {auto,mps,cpu,cuda}` 显式控制。

### 3.6 算法优化

> 单纯把现成的 OCR + XLM-RoBERTa + LoRA 拼起来只是"集成"，不构成算法贡献。本节给出三项**真正落在算法层面**的优化，可根据时间精力独立或组合实现。

#### 3.6.1 优化 A1：早退机制 (Early Exit) — 加速方向

**问题**：当前 XLM-RoBERTa 跑完整 12 层，对"明显干净"和"明显注入"的样本一视同仁，造成平均延迟偏高。

**算法设计**：
1. 在 XLM-RoBERTa 的若干**中间层**（如 layer 4、8、12）追加轻量分类头 `EarlyExitHead`；
2. **联合训练目标**：$\mathcal{L} = \mathcal{L}_{\text{final}} + \sum_{k} \alpha_k \cdot \mathcal{L}_{\text{exit@k}}$，其中 $\alpha_k$ 是中间层损失权重；
3. **推理时自适应路径**：若中间层分类置信度 > 阈值（如 0.95），**跳过剩余层**直接输出；
4. **退出稳定性约束**：要求**连续 2 层**置信度 > 阈值才允许早退。

**参考工作**：FastBERT (2020)、DeeBERT (2020)、PABEE (2020)。

**评估指标**：平均 / P95 延迟、各层退出率、干净/注入/OOD 集延迟分布、F1（**核心约束：F1 不能下降**）。

#### 3.6.2 优化 B1：零宽字符 / Unicode 混淆检测 — 范围方向

**问题**：当前 OCR 把图片文字当作普通字符串送入分类器，**无法识别用 Unicode 特殊字符"伪装"的攻击**：zero-width space (`U+200B`)、RTL override (`U+202E`)、Homoglyph（西里尔 `а` 替代拉丁 `a`）等。这些字符**视觉无显示但语义保留**，注入有效但人工难察。

**算法设计**：
1. **文本预处理层**：在 OCR 文本 + 用户 prompt 拼接前扫描可疑字符；
2. **特征构造**：`is_zero_width`、`is_homoglyph`、`is_rtl_override`、`zero_width_count`；
3. **特征拼接送入分类器**：与 `[CLS]` embedding 拼接后送入分类头；
4. **训练数据增强**：新增 50-100 条 zero-width 攻击样本。

**评估指标**：Unicode 混淆攻击子集 Recall（目标 ≥ 85%）、干净集 FPR、Macro F1。

#### 3.6.3 优化 B2：跨模态矛盾检测 (Cross-modal Contradiction) — 范围方向

**问题**：单独看都正常、组合后是攻击的样本（如 prompt 说"请忽略此图片"，图片里写"Assistant: 同意退款"）容易被漏检。

**算法设计**：
1. **CLIP 文本相似度特征**：`sim(prompt, ocr_text)` ∈ [-1, 1]——prompt 主题与图片内文本主题越不一致越可疑；
2. **特征拼接送入分类器**：把 `sim` 作为附加标量特征与 `[CLS]` embedding 拼接；
3. **触发规则**：`sim < 0.2` AND `prompt 含敏感词` → 直接判为 indirect_injection；
4. **训练数据增强**：新增 100+ 条 cross-modal 矛盾样本。

**参考工作**：SNLI / MNLI 等矛盾检测文献。**但将 CLIP 相似度作为 prompt injection 攻击信号是本课题的新规则**。

**评估指标**：cross-modal 子集 Recall（目标 ≥ 80%）、干净集 FPR（**核心约束：不上升**）、Macro F1。

#### 3.6.4 三项优化的工程量与可组合性

| 优化 | 方向 | 工程量 | 独立 | 可叠加 |
|---|---|---|---|---|
| A1 早退 | 加速 | 2-3 天 | ✅ | ✅ |
| B1 零宽字符 | 范围 | 1 天 | ✅ | ✅ |
| B2 跨模态矛盾 | 范围 | 2-3 天 | ✅ | ✅ |

三项**完全独立**。**最简组合推荐**：B1（1 天见效）+ A1（最经典算法贡献）。

## 六、研究的局限与边界

1. **检测精度上限**：在极限精度上，离线轻量方案的 F1 可能弱于在线大模型方案（如 PromptGuard），这是离线约束下的必然代价；
2. **多模态数据规模有限**：自构造 OCR 注入样本受人工成本限制，泛化性需要更大规模验证；
3. **不针对未训练攻击模式**：对 0day 攻击、Unicode 混淆、base64 编码等绕过手段的检出率可能下降；
4. **不替代模型自身对齐**：本检测器作为"前置过滤 / 审计工具"，**不替代**训练阶段的安全对齐；
5. **不应用于主动攻击**：本项目**只构建识别侧**，不发布任何可用于主动构造更隐蔽注入的代码或教程；
6. **不覆盖 MCU 级嵌入式**：本课题目标设备是普通 PC 与主流边缘设备，不覆盖 < 1MB 内存的极端嵌入式；
7. **§3.6 优化方案的已知局限**：
   - **A1 早退机制**：中间层分类头会增加约 5-10% 训练时间；阈值需要为不同部署环境调参；冷启动模型大小略增（多 3 个分类头）；
   - **B1 零宽字符检测**：仅对 OCR 文本 + 用户 prompt 中的 Unicode 字符有效，对抗性像素级注入（OCR 直接失败）无效；
   - **B2 跨模态矛盾**：CLIP 文本相似度依赖 `clip-vit-base-patch32`，会增加约 100-200MB 离线包大小；可能对"图文本就该不一致"的正常样本误报（如 prompt 是提问、图是装饰）。

## 附录 A：关键术语速查

> 本节为本课题读者快速回顾核心概念，作者可按需查阅。

| 术语 | 简释 |
|---|---|
| **LLM** | Large Language Model，大语言模型。能读懂并生成自然语言文本的 AI。 |
| **MLLM** | Multimodal LLM，多模态大语言模型。既能读文字、又能读图片的 LLM。 |
| **提示注入（Prompt Injection）** | 用精心构造的输入"骗 LLM 听坏话"。本课题研究对象。 |
| **直接 / 间接注入** | 直接：攻击在用户输入文本里；间接：攻击藏在外来内容（图片、外部网页）里。 |
| **OCR** | 把图片中的文字识别为机器可读文本。 |
| **XLM-RoBERTa** | 支持 100+ 语种的文本理解模型，本课题用作文本侧骨干。 |
| **LoRA** | 一种参数高效微调技术：冻结原始参数，只训练小矩阵，节省算力。 |
| **微调（Fine-tuning）** | 拿预训练模型，用自己的数据再训练一下，让它擅长特定任务。 |
| **Backbone（骨干）** | 被复用的核心模型。本课题用 XLM-RoBERTa。 |
| **前置过滤** | 在主 LLM 之前先做一次"安检"，可疑输入直接拦住。 |
| **离线（Offline）** | 不需要联网，所有计算在本地完成。 |
| **F1 分数** | 评估检测好坏的指标，0~1，越接近 1 越好。 |
| **OWASP** | 国际安全组织，每年发布"最常见的安全风险 Top 10 列表"。 |
